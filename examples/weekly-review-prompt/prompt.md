# Weekly Review Prompt

Every Friday at 17:00, open a new tab with a pre-filled weekly review template. Turns "I should reflect on the week" from a TODO into a thing that just appears on your screen.

## Trigger

On each scheduled run:

1. Check day-of-week and time. Only proceed if Friday and time is 17:00-17:59 local.
2. `storage_get` `lastWeeklyReview`. Skip if it's already today.

## Do the thing

1. Build the review template based on the week (Monday to Friday). Write it to `OPFS://reviews/YYYY-MM-DD.md`:

   ```markdown
   # Weekly review — week of <Monday's date>

   ## What went well
   -

   ## What didn't
   -

   ## What I learned
   -

   ## Next week's top 3
   1.
   2.
   3.

   ## Week in browsing
   <auto-generated from history_search — 3 bullet points summarising the week's themes>
   ```
2. `tab_open` a new tab to `chrome://new-tab` (or data: URL with the template as HTML if supported).
3. `notification_show`: "Friday 5pm — time for a weekly review. Template saved to OPFS."
4. `storage_set` `lastWeeklyReview: today`.

## Constraints
- Once per week. Never on Saturday even if Friday was missed.
- Use `history_search` to add the "Week in browsing" section. Filter to Mon-Fri 00:00 of this week.

## Required permissions
`tabs`, `history`, `notifications`.
